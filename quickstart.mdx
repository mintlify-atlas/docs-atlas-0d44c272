Mercury takes about 30 seconds to install and configure. You need Node.js 20 or later and an API key from at least one LLM provider — DeepSeek is a good starting point because it's cost-effective and has a generous free tier. This guide walks you from a fresh machine to a running agent.

## Install and start Mercury

<Steps>
  <Step title="Install Mercury">
    Run Mercury directly with `npx` (no install required), or add it globally so you can use the `mercury` command from anywhere:

    <CodeGroup>
    ```bash npx (no install)
    npx @cosmicstack/mercury-agent
    ```

    ```bash Global install
    npm i -g @cosmicstack/mercury-agent
    mercury
    ```
    </CodeGroup>

    <Tip>
      Use `npx` to try Mercury without committing to a global install. You can always install globally later with `npm i -g @cosmicstack/mercury-agent`.
    </Tip>
  </Step>

  <Step title="Complete the setup wizard">
    On first run, Mercury detects that no configuration exists and launches the setup wizard automatically. It asks for:

    - **Your name** — used in Mercury's identity and memory
    - **An LLM API key** — at minimum, one provider key is required (DeepSeek, OpenAI, Anthropic, Grok, or Ollama)
    - **A Telegram bot token** (optional) — skip this for now if you only want the CLI

    Press Enter to accept a suggested value, or type your own.

    ```
    Welcome to Mercury Agent setup

    What's your name? › Alice
    Default LLM provider [deepseek]: › deepseek
    DeepSeek API key: › sk-xxxxxxxxxxxxxxxx
    Telegram bot token (optional, press Enter to skip): ›

    Configuration saved to ~/.mercury/mercury.yaml
    ```

    Setup completes in under 30 seconds. Your config is saved to `~/.mercury/mercury.yaml` and your API keys to `~/.mercury/.env`.
  </Step>

  <Step title="Start the agent">
    After setup, Mercury starts automatically. In future sessions, run:

    ```bash
    mercury
    ```

    or equivalently:

    ```bash
    mercury start
    ```

    You'll see the agent initialize and a prompt appear:

    ```
    Mercury is ready. Type a message or /help for commands.
    > 
    ```
  </Step>

  <Step title="Send your first message">
    Type any message and press Enter. Mercury will respond and, if needed, ask for permission before taking any action on your filesystem or shell.

    ```
    > What files are in my home directory?

    Mercury: I'll take a look.
    ⚠ Mercury needs read access to ~/. Allow? (y/n/always): always
    [Scope saved to ~/.mercury/permissions.yaml]

    Here's what's in your home directory: ...
    ```

    Once you grant a scope with `always`, Mercury remembers it and never asks again.
  </Step>

  <Step title="Use in-chat commands">
    These commands work during a conversation and don't consume API tokens:

    | Command | Description |
    |---------|-------------|
    | `/help` | Show the full manual |
    | `/status` | Show config, budget, and usage |
    | `/tools` | List all loaded tools |
    | `/budget` | Check token budget status |
    | `/budget reset` | Reset daily usage to zero |
    | `/budget set <n>` | Change daily token budget |
  </Step>
</Steps>

## Run Mercury as a background daemon

To keep Mercury running 24/7 — with auto-restart on crash and auto-start on boot — use `mercury up`:

```bash
mercury up
```

This installs the system service (if not already installed) and starts the background daemon. If Mercury is already running, it confirms that and shows the PID.

```bash
mercury restart   # Restart the background process
mercury stop      # Stop the background process
mercury logs      # View recent daemon logs
mercury status    # Show whether the daemon is running
```

<Note>
  In daemon mode, Telegram becomes your primary interface. The CLI is log-only since there's no terminal for input. See [daemon mode](/daemon/overview) for details.
</Note>

## Reconfigure at any time

Changed your API key, want to add a Telegram bot, or just need to update settings? Run:

```bash
mercury doctor
```

The doctor command walks you through all settings interactively. Press Enter to keep any current value — you only need to type where you want to make a change.

```bash
mercury setup   # Re-run the full first-time setup wizard
mercury status  # Show current configuration and daemon status
```
